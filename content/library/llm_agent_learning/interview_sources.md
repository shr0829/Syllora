# 大模型应用开发 / Agent 开发面经与题单索引

用途说明：

- 这份索引主要用于“挖高频面试问题”
- 其中大部分是社区题单、博客、面经复盘，不是官方标准答案
- 真正用于学习和实现时，仍然应该优先查官方文档、论文和主仓库源码

## 真实面经收录规则

从这次开始，凡是放进“真实面经”分区的条目，必须满足至少一个条件：

- 标题或正文明确写明是一面 / 二面 / 实习 / 校招 / 社招复盘
- 能看出公司、岗位、轮次中的至少一项
- 如果是知乎转载，正文里必须能看到来源声明或明确是个人真实复盘

以下内容不计入“真实面经”：

- 八股整理
- 题单汇总
- 教程文章
- 课程导流
- 没有任何面试上下文的知识贴

## 来源优先级

后续补充来源时，优先顺序按这个来：

1. `牛客网`：真实面经密度最高，最适合抽真实高频问题
2. `知乎`：适合收集面经整理、专题归纳、路线总结
3. `公众号`：适合收集成体系的专题题单和答题框架
4. 其他博客 / 聚合站：适合作为补充，不作为第一优先级

使用原则：

- 用 `牛客网` 提取“真实被问到什么”
- 用 `知乎` 和 `公众号` 提取“这些问题通常怎么归类”
- 用 `official_references.md` 校正答案，不直接迷信题单答案

## 从这些来源里能抽出的高频问题

下面这些不是逐字摘录，而是对高频问题的归纳：

1. `RAG` 和 fine-tuning 的适用边界是什么？
2. chunking 为什么重要？你如何选 chunk size 和 overlap？
3. embedding model、reranker、vector DB 各负责什么？
4. 如何定位 `RAG` 错误是发生在召回阶段还是生成阶段？
5. `ReAct` 是什么？和普通 workflow / chain 的区别是什么？
6. function calling / tool schema 设计为什么会影响 Agent 稳定性？
7. 什么时候应该用 `LangGraph`，而不是简单 `LangChain`？
8. 多智能体相比单智能体的收益和代价分别是什么？
9. 你如何给 Agent 系统做 eval、trace、cost control、guardrails？
10. `LoRA` / `QLoRA` / `DPO` / `RLHF` 各自解决什么问题？

## 如何使用这份索引

建议顺序：

1. 先看 A、B、C 三部分，建立应用开发和 `RAG` / Agent 基础概念
2. 再看 D、E，开始补系统设计和生产化视角
3. 最后看 F，补微调与后训练

不要把这份索引当成“背题库”。

正确用法是：

- 先用它发现高频问题
- 再回到官方文档和代码把每个问题做成可运行 demo
- 最后用自己的项目回答这些题
